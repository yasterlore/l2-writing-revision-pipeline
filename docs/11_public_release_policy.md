# Public Release Policy

Before public release:

- Choose and apply the final license.
- Confirm no real participant data is present.
- Confirm all examples and fixtures are synthetic.
- Run repository-wide secret scanning if available.
- Run formatting, linting, tests, and dependency/security audits if available.
- Review documentation for no-oracle compliance.
- Review `.gitignore` and release packaging rules.

Do not publish datasets, logs, screenshots, or derived artifacts that may contain real participant data unless they have passed institution-approved release review.
